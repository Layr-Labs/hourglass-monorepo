# Hourglass Kubernetes Operator - Implementation Milestones

## üîÑ **ARCHITECTURE CHANGE: Singleton Operator + User-Managed Executors**

**New Architecture**: Single Hourglass Operator instance manages `Performer` CRDs cluster-wide. Users deploy multiple Executors as StatefulSets independently.

**Benefits**: 
- ‚úÖ Clean separation of concerns (no circular dependencies)
- ‚úÖ Singleton operator scales to handle multiple AVS executors
- ‚úÖ Each executor deployed independently with flexible configuration
- ‚úÖ Proper StatefulSet semantics for Executor persistence and scaling
- ‚úÖ Greater flexibility for production deployments
- ‚úÖ Simpler operator focused on core responsibility
